# slate-faculty-interview-portal-starter
This repository is a simple, practical guide for building a **faculty-friendly interview process in Slate** using portals, events, scheduler, and evaluation forms.
## Who this is for
- Graduate and professional programs  
- Teams working with faculty interviewers  
- Schools using (or receiving data from) systems like PharmCAS  
- Slate users who want to reduce spreadsheets, emails, and manual coordination  

## What you can build
Using the steps in this repo, you can create:
- A **faculty portal** (no full Slate access required)
- A **structured interview day workflow**
- **Application-tied evaluation forms**
- A process that keeps **all data inside Slate**

## Why this matters
Many interview processes are split across:
- Slate
- spreadsheets
- email
- shared drives

That fragmentation creates:
- duplicate work  
- missing data  
- poor visibility for admissions and faculty  

This approach brings everything back into **one coordinated workflow**.

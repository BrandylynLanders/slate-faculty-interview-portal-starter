# 2. Build Your Foundation

Before building the portal, set up the pieces that support it.

---

## 1. Identify your faculty users

You need:
- A list of faculty interviewers  
- Their roles (interviewer, chair, etc.)  
- Any grouping (department, unit, location)

---

## 2. Add key fields to support your process

Common useful fields:

On the **user record**:
- Faculty Unit  
- User Group  
- Location  
- Interviewer Role  

These allow you to:
- control access  
- filter views  
- assign interviews  

---

## 3. Decide: User vs Dataset

Simple guidance:

Use **User-based setup** if:
- Faculty are internal users  
- You want simpler access control  
- You want to avoid duplicate data  

Use a **Dataset** if:
- You need to store more complex external info  
- Faculty are not users in Slate  

Start simple if possible.

---

## 4. Application-level support

Consider adding fields such as:
- Interviewer Assigned  
- Interview Result  

These help with:
- tracking  
- reporting  
- future scalability  

---

## 5. Roles and access

Create a role such as:
- “Faculty Interviewer”

Use this to:
- restrict portal access  
- control visibility  

---

## Key idea

You are not building everything yet.

You are creating:
> the structure that allows the portal to work cleanly
